# 05 — API Response Shapes (field-level map)

> **Purpose:** authoritative, field-level map of **every** HTTP endpoint exposed by the
> FastAPI backend at `cashflow-web/backend/`. This is the source of truth for the React
> frontend's snake_case → component mapping layer.
>
> **Generated by reading:** all `app/api/routers/*.py`, `app/api/{main,auth,deps,errors,export,planning,audit,schemas}.py`,
> `app/db/models.py`, the `app/domain/{allocation,forecast}.py` helpers, and the test
> suites under `tests/api/` (which assert the literal JSON keys/types).
>
> **Conventions used below**
> - `(verify against live)` = field exists but its concrete value/derivation depends on domain
>   logic or seeded data; shape is fixed, value is not asserted by a test.
> - All money fields are `float` in JSON (`Decimal` cast to `float` at the router boundary).
> - All timestamps are ISO-8601 strings (Pydantic serializes `datetime`/`date`).
> - All keys are **snake_case**.

---

## 0. Cross-cutting facts

### 0.1 Auth model
- **Cookie:** HttpOnly session cookie named **`session`** (Starlette `SessionMiddleware`,
  `same_site="lax"`, `https_only=False` for LAN). Set on successful `POST /api/auth/login`.
- **Every endpoint requires the session cookie EXCEPT:**
  - `GET /api/health`
  - `POST /api/auth/login`
  - `POST /api/auth/logout`
  - (`GET /api/auth/me` **does** require auth → 401 without cookie.)
- Missing/invalid/inactive session → **401** with the unified error envelope and
  `error.code == "unauthorized"`, `error.message == "يجب تسجيل الدخول"` (Arabic).
- Auth is enforced via the `get_current_user` dependency. The dependency is wired as
  `_user=Depends(get_current_user)` on read endpoints and `user=Depends(...)` on writes.

### 0.2 Unified error envelope (`app/api/errors.py`)
Every error response (401/404/405/409/422/429/500 — anything raised via `ApiError`,
`HTTPException`, `RequestValidationError`, or an unhandled exception) shares this shape:

```jsonc
{
  "error": {
    "code":    "string",   // machine-readable, see table below
    "message": "string"    // human-readable (often Arabic)
  }
}
```

| HTTP | `error.code` | When |
|------|--------------|------|
| 400 | `bad_request` | generic |
| 401 | `unauthorized` | not logged in / inactive user / bad credentials |
| 403 | `forbidden` | (mapping exists; not currently raised) |
| 404 | `not_found` | resource missing |
| 405 | `method_not_allowed` | wrong verb |
| 409 | `conflict` | duplicate baseline scenario, duplicate cap (supplier_id+effective_from), duplicate plan (year_month+scenario_id), duplicate assumptions, delete with dependents |
| 409 | `etl_running` | `POST /api/etl/run` while a run is in progress (note: **not** `conflict`) |
| 422 | `validation_error` | Pydantic body/query validation. Message = `"<loc>: <msg>"` (loc strips the leading `body`). |
| 429 | `too_many_requests` | login throttle (5 failures / 5 min per username) |
| 500 | `internal_server_error` | unhandled exception (message is generic — never leaks internals) |

> **Frontend note:** the 422 envelope is **flattened** — it does NOT return FastAPI's default
> `detail: [{loc,msg,type}]` array. It returns the unified `{error:{code,message}}` with a
> single human-readable string. Do not parse a `detail` array.

## 1. ops + auth

### `GET /api/health`
- **Auth:** none.
- **200:**
```jsonc
{ "status": "ok" }
```

### `POST /api/auth/login`
- **Auth:** none (this is how you get the cookie).
- **Body:**
```jsonc
{ "username": "string", "password": "string" }
```
- **200** (also sets the `session` cookie):
```jsonc
{
  "username":     "string",
  "display_name": "string | null"
}
```
- **401** `unauthorized` — unknown user, inactive, or wrong password.
- **429** `too_many_requests` — 6th attempt within 5 min after 5 failures (per username, lowercased).

### `POST /api/auth/logout`
- **Auth:** none (clears whatever session exists).
- **200:**
```jsonc
{ "status": "ok" }
```

### `GET /api/auth/me`
- **Auth:** **required** (401 without cookie).
- **200:**
```jsonc
{
  "username":     "string",
  "display_name": "string | null"
}
```
> Note: neither login nor `/me` ever serialize `password_hash` (asserted by test).

```jsonc
}
```
- `last_etl` is **`null`** when no ETL run rows exist.

## 4. cashflow router — `GET /api/cashflow/monthly`

- **Auth:** required.
- **Query params:**
  - `perspective` = `comprehensive` (default) | `operational` — a `Literal`; any other value → **422**.
- **Perspective behaviour** (per-month `out_total_m` / `net_total_m`):
  - `comprehensive`: `out_total_m = out_total_comprehensive_m`, `net_total_m = net_total_m` (includes siyrafa).
  - `operational`: `out_total_m = out_total_operational_m`, `net_total_m = net_operating_m` (excludes siyrafa).
- **200** (`CashflowMonthlyResponse`):
```jsonc
{
  "months": [                            // list — ALL monthly_cashflow rows, ascending
    {
      "year_month":     "2022-05",       // str
      "cash_in_m":      110.0,           // float
      "out_total_m":    100.0,           // float — PERSPECTIVE-AWARE
      "net_total_m":    10.0,            // float — PERSPECTIVE-AWARE
      "cash_running_m": 70.0,            // float
      "fiscal_year":    "2022-2023"      // str
    }
  ],
  "forecast": [                          // list — pivoted forecast_base (engine='seasonal'); may be []
    {
      "year_month":  "2026-05",          // str
      "cash_in_m":   0.0,                // float — series 'cash_in'
      "out_total_m": 0.0,                // float — Σ of OUT series keys (perspective-aware: operational excludes 'out_siyrafa')
      "net_total_m": 0.0                 // float — cash_in_m − out_total_m
    }
  ],
  "by_fiscal_year": [                    // list — aggregated per fiscal_year, sorted ascending
    {
      "fiscal_year": "2022-2023",        // str
      "in_m":  1150.0,                   // float
      "out_m": 985.0,                    // float — perspective-aware sum
      "net_m": 165.0                     // float — perspective-aware sum
    }
  ]
}
```
- Forecast OUT series keys: comprehensive = `{out_suppliers, out_drawings, out_refunds, out_purchases, out_salaries, out_other, out_siyrafa}`; operational drops `out_siyrafa`.

```jsonc
{
  "expense_cats": [                      // list — EXACTLY 6, ordered: suppliers, partners, siyrafa, purchases, salaries, refunds
```

## 6. suppliers router

### `GET /api/suppliers`
- **Auth:** required.
- **Query params:** none.
- **200** (`SuppliersResponse`): suppliers ordered by `display_order` ascending.
```jsonc
{
  "suppliers": [
    {
      "id":        1001,        // int — this is the SUPPLIER ACCOUNT_ID (suppliers.account_id), NOT the suppliers.id PK
      "name":      "معرض البركة", // str
      "cap":       5.0,          // float — active SupplierCap.monthly_cap_m (greatest effective_from <= today), else 0.0
      "currency":  "IQD",        // str — suppliers.currency or "IQD"
      "monthly":   [2.8, 2.2],   // list[float] — last-12 per_supplier_monthly.paid_m, ascending (len 0..12)
      "over_cap":  1,            // int — count of months where paid_m > cap (only when cap>0; else 0)
      "balance_m": 0.0,          // float — latest supplier balances_snapshot.balance_iqd_m, else 0.0
      "util":      0.71,         // float | null — avg(monthly)/cap when cap>0 and monthly non-empty, else null
      "active":    true          // bool
    }
  ]
}
```

### `POST /api/suppliers/{account_id}/caps` (WRITE)
- **Auth:** required.
- **Path:** `account_id` (int) = supplier's `account_id` (e.g. 1001).
- **Body** (`CapCreate`):
```jsonc
{
  "monthly_cap_m":  12.0,        // float, required, >= 0
  "effective_from": "2026-06-01", // date "YYYY-MM-DD", required
  "plan_low_m":     0.0,         // float, default 0.0, >= 0
  "plan_high_m":    0.0,         // float, default 0.0, >= 0
  "user_monthly_m": 0.0          // float, default 0.0, >= 0
}
```
- Validation: if `plan_low_m > 0` and `plan_high_m > 0` and `plan_low_m > plan_high_m` → **422**.
- **201** (`CapOut`):
```jsonc
{
  "id":             1,           // int — supplier_caps.id PK
  "supplier_id":    1,           // int — suppliers.id PK (NOT account_id)
  "monthly_cap_m":  12.0,        // float
  "plan_low_m":     0.0,         // float
  "plan_high_m":    0.0,         // float
  "user_monthly_m": 0.0,         // float
  "effective_from": "2026-06-01", // date
  "created_by":     1            // int | null — users.id
}
```
- **404** `not_found` — no supplier with that `account_id`.
- **409** `conflict` — a cap for (supplier_id, effective_from) already exists.
- **422** — negative cap or invalid plan range.
- Writes a `create_cap` audit row in the same transaction.

## 8. forecast router — `GET /api/forecast`

- **Auth:** required.
- **Query params:**
  - `scenario_id` (int, optional). If given and a Scenario row + per-scenario `assumptions.income_growth_pct` exist, that overrides the global income-growth multiplier. A non-existent `scenario_id` does **not** error (falls back to global; returns 200).
- Three standard scenarios are always returned: `base` (label `متحفّظ`, in_g 1.00, out_g 1.00),
  `opt` (`متفائل`, 1.08, 0.98), `pess` (`متشائم`, 0.92, 1.06).
- **200** (`ForecastResponse`):
```jsonc
{
  "forecast": [                          // list — one per forecast month (ascending year_month), 12 with a full horizon
    {
      "year_month": "2026-05",           // str
      "base": { "in_m": 0.0, "out_m": 0.0, "net_m": 0.0 },  // floats; net = in − out
      "opt":  { "in_m": 0.0, "out_m": 0.0, "net_m": 0.0 },
      "pess": { "in_m": 0.0, "out_m": 0.0, "net_m": 0.0 }
    }
  ],
  "cash_paths": {                        // dict keyed by scenario → running cash list (one float per month, reserve subtracted each month)
    "base": [0.0],                       // list[float]
    "opt":  [0.0],
    "pess": [0.0]
  },
  "fc_totals": {                         // dict keyed by scenario → aggregated totals
    "base": {
      "in_m":       0.0,                 // float — Σ over horizon
      "out_m":      0.0,                 // float
      "net_m":      0.0,                 // float
      "end_cash_m": 0.0,                 // float — last point of cash_paths[scenario]
      "min_cash_m": 0.0                  // float — min of cash_paths[scenario]
    },
    "opt":  { ... },
    "pess": { ... }
  },
  "scenarios": {                         // dict keyed by scenario → metadata
    "base": { "label": "متحفّظ", "in_g": 1.0,  "out_g": 1.0  },  // str, float, float
    "opt":  { "label": "متفائل", "in_g": 1.08, "out_g": 0.98 },
    "pess": { "label": "متشائم","in_g": 0.92, "out_g": 1.06 }
  },
  "mape":       18.0,                    // float | null — cash_in series MAPE from forecast_base
  "confidence": "عالية"                  // str | null — عالية (<25) | متوسطة (<50) | منخفضة (>=50); null when mape null
}
```
- `mape`/`confidence` are `null` when no MAPE is present on the `cash_in` forecast series.

## 9. supplier_plan router — `GET /api/supplier-plan`

- **Auth:** required.
- **Query params:**
  - `month` (str, **required**) — pattern `^\d{4}-(0[1-9]|1[0-2])$` (`YYYY-MM`). Missing → 422; malformed (e.g. `2026-13`) → 422.
  - `scenario_id` (int, optional) — accepted for API symmetry; not yet used.
- Pool = `forecast_in − salaries − purchases − refunds − partners − siyrafa − reserve` (Option-1: dollar suppliers funded via siyrafa, get `allocated_m = 0`).
- **200** (`SupplierPlanResponse`):
```jsonc
{
  "month":   "2026-05",          // str — echoes the request
  "pool_m":  0.0,                // float — distributable dinar pool (>= 0)
  "alloc": [                     // list — one per supplier (active, ordered by display_order)
    {
      "id":            1001,     // int — supplier ACCOUNT_ID (not the PK)
      "name":          "string", // str
      "currency":      "IQD",    // str — "USD" entries always have allocated_m == 0
      "allocated_m":   0.0,      // float — dinar allocated (rounded to 6dp)
      "actual_paid_m": 0.0       // float | null — per_supplier_monthly.paid_m for this month, else null
    }
  ],
  "leftover_m": 0.0              // float — un-allocated remainder (>= 0)
}
```
- The underlying `allocate_dinar` also computes a `distributed` total internally, but that is
  **not** exposed in this endpoint's JSON (it is folded into `leftover_m = pool − distributed`).

## 10. scenarios router

### `GET /api/scenarios`
- **Auth:** required. **200** = `list[ScenarioOut]` (ordered by `id` ascending):
```jsonc
[
  {
    "id":          1,            // int
    "name":        "string",     // str
    "kind":        "string|null", // str | null
    "is_baseline": false,        // bool
    "description": "string|null"  // str | null
  }
]
```

### `POST /api/scenarios` (WRITE)
- **Auth:** required.
- **Body** (`ScenarioCreate`):
```jsonc
{
  "name":        "string",       // str, required
  "kind":        "string | null", // optional, default null
  "is_baseline": false,          // bool, default false
  "description": "string | null"  // optional, default null
}
```
- **201** → `ScenarioOut` (above).
- **409** `conflict` — `is_baseline=true` when a baseline already exists.
- Audit: `create_scenario`.

### `PUT /api/scenarios/{scenario_id}` (WRITE)
- **Auth:** required. **Path:** `scenario_id` (int).
- **Body** (`ScenarioUpdate`, all optional — only non-null fields applied):
```jsonc
{ "name": "string|null", "kind": "string|null", "is_baseline": true|false|null, "description": "string|null" }
```
- **200** → `ScenarioOut`.
- **404** `not_found`; **409** `conflict` (set baseline when another baseline exists).
- Audit: `update_scenario`.

### `DELETE /api/scenarios/{scenario_id}` (WRITE)
- **Auth:** required.
- **200:**
```jsonc
{ "deleted": 5 }               // int — the deleted scenario_id
```
- **404** `not_found`; **409** `conflict` (dependent rows, e.g. a `scenario_adjustments` FK).
- Linked `assumptions` rows are pre-deleted; remaining FK dependents cause 409.
- Audit: `delete_scenario`.

### `PUT /api/scenarios/{scenario_id}/assumptions` (WRITE — upsert)
- **Auth:** required.
- **Body** (`AssumptionUpdate`, all optional; only provided fields written):
```jsonc
{
  "usd_rate":                null,  // float | null, >= 0
  "unexpected_reserve_m":    null,  // float | null, >= 0
  "income_growth_pct":       null,  // float | null
  "in_growth_factor":        null,  // float | null
  "out_growth_factor":       null,  // float | null
  "cagr_floor":              null,  // float | null
  "cagr_cap":                null,  // float | null
  "forecast_horizon":        null,  // int   | null, >= 1
  "fiscal_year_start_month": null,  // int   | null, 1..12  (13 → 422)
  "forecast_engine":         null   // str   | null
}
```
- **200** (`AssumptionOut`):
```jsonc
{
```

### 17.12 General rules the mapping layer should encode
1. **snake_case → camelCase** for every multi-word key (`out_total_m` → `outTotalM`, etc.).
2. **Strip/normalize the `_m` suffix** (everything is already in millions; UI naming may drop `_m`).
3. **`in`/`out`/`net` are always suffixed `_m`** in the API and nested under scenario keys in forecast.
4. **`id` in supplier/alloc responses = account_id**, but write responses (`CapOut.supplier_id`,
   `PaymentPlanLineOut.supplier_id`) use the **suppliers PK** — keep both identities distinct.
5. **Watch the spelling `siyrafa` (API) vs `sayrafa` (data.js)** and `not_due` (API) vs `current` (data.js).
6. **Two names for one concept across endpoints:** reserve (`reserve_m` vs `unexpected_reserve_m`),
   fiscal-year start (`fy_start` vs `fiscal_year_start_month`), monthly OUT
   (`out_total_m` vs `out_total_comprehensive_m`).
7. **Error handling:** always read `error.code` + `error.message` from the unified envelope; the
   429 throttle, 409 `etl_running`, and flattened 422 are the non-obvious cases.
