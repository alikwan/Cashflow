## النموذج الصحيح (من `MssqlDriver`)
الجداول الستة المعتمدة: `accountTypes, accounts, Premiums, PremiumPays, Bonds, Bills`.

- `Premiums` = **خطة لكل فاتورة** (`PremiumPayAmount` = قيمة القسط الاسمية، `TotalAmount`، `BillId`).
- **`PremiumPays` = صف لكل قسط** (الجدولة الفعلية)، وفيه:
  - `Date` = **تاريخ استحقاق القسط** (موجود صراحةً — لا اشتقاق).
  - `DatePay` = تاريخ الدفع (`<= '1900-01-02'` = لم يُدفع).
  - **`Amount` = الرصيد المتبقّي للقسط** (ليس المدفوع): غير مدفوع → `= PremiumPayAmount`؛ مدفوع كاملاً → `0`؛ جزئي → المتبقّي؛ غرامة → `> PremiumPayAmount`.
  - `Discount`، `PremiumState` (3,4 = مخصوم/مسوّى).
- **النقد المدفوع للقسط** = `MAX(0, PremiumPayAmount − Amount − Discount)` عند ضبط `DatePay`.
- **مسوّى (settled)** = `PremiumState ∈ (3,4)` أو (`Amount = 0` و `DatePay` مضبوط).
- **الرصيد القائم للقسط** = إن لم يُسوَّ → `Amount`؛ وإلا → `0`.

## مواصفة `installments_summary` / `installments_aging` (مصحّحة)
- `installments_summary`: `remaining_m` = الرصيد القائم (Σ القائم غير المسوّى) ≈ 1.26 مليار؛ + `cash_paid_m` ≈ 5.65 مليار، `discount_m` ≈ 0.15 مليار، `face_total_m` ≈ 7.12 مليار.
- `installments_aging`: لكل قسط **غير مسوّى وقائم>0**، الشريحة من `pp.Date` مقابل اليوم؛ `bucket_key ∈ {not_due, b0_30, b31_60, b61_90, b91_120, b120}` (+ `stale` اختياري لعقود بلا دفعة >12 شهر — قرار المالك "المزيج"، يُطبَّق على القائم الصحيح).
- ETL يفرض Σ(amount عبر الشرائح) = `installments_summary.remaining_m`.
